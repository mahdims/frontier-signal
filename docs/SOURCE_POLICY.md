# Source and privacy policy

## Source classes

### Primary technical evidence

- paper or technical report;
- official repository;
- model weights;
- benchmark code and data;
- release notes;
- reproducible independent test.

### Impact evidence

- deployment;
- cloud/platform integration;
- downstream package adoption;
- credible technical discussion;
- job-market demand;
- customer use;
- event/workshop activity;
- repository contributor velocity.

### Discovery-only evidence

- technical media;
- executive post;
- social engagement;
- funding announcement;
- screenshots without a public artifact.

Discovery-only evidence can trigger investigation but cannot establish technical truth.

## Private and restricted sources

The system may store a private-group item for personal use, but:

- it must be marked `visibility=private`;
- it must not expose member identities;
- it must not reproduce private text in a shareable report;
- it must not pretend the content has a public URL;
- it must be corroborated before being treated as fact.

## Platform compliance

Use official APIs where available. Use RSS or public pages where permitted. Use manual forwarding, exports, or user-operated browser capture for closed platforms. Do not bypass authentication, access controls, rate limits, or anti-bot protections.
